Accessibility by default.

## Commitment

Reliverse is built to be usable by everyone. Accessibility is part of the design process, not an afterthought.

## Principles

- **Keyboard navigation**: Full support for keyboard-only interaction.
- **Screen readers**: Semantic markup and ARIA where needed.
- **Focus management**: Visible focus indicators and logical focus flow.
- **Color and contrast**: Accessible color choices and sufficient contrast.
- **Calm microcopy**: Clear language without urgency mechanics.

## Ongoing Work

Accessibility passes are part of the development process for web, mobile, and desktop apps. As features ship, they are reviewed for accessibility.

If you encounter accessibility barriers, [report them](https://github.com/reliverse/reliverse/issues).
